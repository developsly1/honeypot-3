Web Image Formats: A Quick Reference

This note collects the rules of thumb we use when deciding which image format to ship for a given asset. Picking the right format is usually the largest single lever on page weight, ahead of most other front-end optimizations.

Photographs

For photographic content with smooth gradients, JPEG remains a safe default, but modern formats compress the same image substantially smaller at equivalent quality. WebP typically saves a quarter to a third over JPEG, and AVIF often goes further still, at the cost of slower encoding. Serving a modern format with a JPEG fallback gives the best of both.

Graphics and transparency

PNG is lossless and supports transparency, which makes it the right choice for logos, screenshots, and flat graphics where crisp edges matter. For the same images, WebP and AVIF also support transparency and usually produce smaller files, so they are worth testing before defaulting to PNG.

Vectors

SVG describes shapes mathematically, so it scales to any size without loss and stays tiny for icons and simple illustrations. Because it is text, it can be minified, inlined, and styled with CSS. Avoid SVG for photographic content, where it offers no benefit.

Delivery

Whatever the format, set explicit width and height so the layout does not shift while images load, and lazy-load anything below the fold. Responsive sources let the browser pick an appropriately sized file for the viewport rather than downloading a single large original everywhere.

Reference notes only. Measure with your own assets; results vary by image content.
